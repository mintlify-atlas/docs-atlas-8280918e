Model types define the different AI/ML providers supported by Syft Space. Each type has its own configuration requirements and capabilities.

## List model types

Get all available model types with their configuration schemas.

### Response

<ResponseField name="name" type="string">
  Model type identifier used in the `dtype` field when creating models.
</ResponseField>

<ResponseField name="description" type="string">
  Human-readable description of the model type and its capabilities.
</ResponseField>

<ResponseField name="icon" type="string">
  Icon representation for the model type.
</ResponseField>

<ResponseField name="enabled" type="boolean">
  Whether this model type is currently enabled. Disabled types may lack required dependencies.
</ResponseField>

<ResponseField name="config_schema" type="object">
  JSON schema describing the required and optional configuration fields for this model type.
</ResponseField>

<RequestExample>
```bash cURL
curl https://your-server.com/api/v1/models/types \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```python Python
import requests

response = requests.get(
    "https://your-server.com/api/v1/models/types",
    headers={"Authorization": "Bearer YOUR_API_KEY"}
)

types = response.json()
for model_type in types:
    print(f"{model_type['name']}: {model_type['enabled']}")
```
</RequestExample>

## Supported model types

Syft Space supports the following model types:

### OpenAI

**Type identifier:** `openai`

Supports OpenAI's chat completion API and OpenAI-compatible APIs (e.g., Azure OpenAI, LocalAI, LM Studio).

**Configuration:**
- `api_key` (required): OpenAI API key
- `model`: Model identifier (default: `gpt-3.5-turbo`)
- `base_url`: Custom API endpoint (default: `https://api.openai.com/v1`)
- `system_prompt`: Default system message

**Example models:** `gpt-4`, `gpt-4-turbo`, `gpt-3.5-turbo`

### Anthropic

**Type identifier:** `anthropic`

Supports Anthropic's Claude models via their API.

**Configuration:**
- `api_key` (required): Anthropic API key
- `model`: Model identifier (e.g., `claude-3-opus-20240229`)
- `max_tokens`: Maximum response length

**Example models:** `claude-3-opus`, `claude-3-sonnet`, `claude-3-haiku`

### Ollama

**Type identifier:** `ollama`

Supports locally-hosted Ollama models.

**Configuration:**
- `base_url` (required): Ollama server URL (e.g., `http://localhost:11434`)
- `model` (required): Model name installed in Ollama

**Example models:** `llama2`, `mistral`, `codellama`, `phi`

### vLLM

**Type identifier:** `vllm`

Supports vLLM inference server for high-performance model serving.

**Configuration:**
- `base_url` (required): vLLM server URL
- `model` (required): Model identifier
- `api_key`: Optional authentication token

<Note>
Model type availability depends on installed dependencies. Check the `enabled` field when listing types to see which are currently available in your deployment.
</Note>
