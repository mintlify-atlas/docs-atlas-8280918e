---
title: Create dataset
description: Create a new dataset with specified configuration
---

Create a new dataset with a specific type and configuration. For file-ingestable dataset types, ingestion automatically starts after creation.

## Request

<ParamField path="POST" type="string">
  `/datasets/`
</ParamField>

### Authentication

Requires authentication via tenant credentials.

### Request body

<ParamField body="name" type="string" required>
  Unique name for the dataset (must be unique per tenant)
</ParamField>

<ParamField body="dtype" type="string" required>
  Dataset type name. Available types:
  - `local_file` - Local filesystem with ChromaDB for vector search
  - `remote_weaviate` - Remote Weaviate server connection
</ParamField>

<ParamField body="configuration" type="object" required>
  Configuration schema specific to the dataset type. Use the [list dataset types](/api/datasets/types) endpoint to get the schema for each type.

  **For local_file:**
  - `collectionName` (string) - ChromaDB collection name (alphanumeric and underscores only)
  - `httpPort` (integer) - ChromaDB server HTTP port (default: 8100)
  - `ingestionPaths` (array) - List of file/directory paths to watch and ingest
  - `fileTypes` (array) - Allowed file extensions (e.g., [".pdf", ".txt", ".md"])

  **For remote_weaviate:**
  - `http_url` (string) - HTTP URL of the Weaviate server
  - `grpc_url` (string) - gRPC URL of the Weaviate server
  - `api_key` (string) - API key for authentication
  - `collection_name` (string) - Name of the Weaviate collection
  - `headers` (object, optional) - Additional headers for third-party API keys
  - `content_property` (string, optional) - Property name to use as main content
  - `metadata_properties` (array, optional) - Properties to include in metadata
  - `filters` (object, optional) - Filter conditions for search queries
</ParamField>

<ParamField body="summary" type="string" default="">
  Brief summary describing the dataset
</ParamField>

<ParamField body="tags" type="string" default="">
  Comma-separated tags (e.g., "legal,documents,analysis")
</ParamField>

## Response

<ResponseField name="id" type="string">
  Unique identifier (UUID) for the dataset
</ResponseField>

<ResponseField name="name" type="string">
  Dataset name
</ResponseField>

<ResponseField name="dtype" type="string">
  Dataset type name
</ResponseField>

<ResponseField name="configuration" type="object">
  Full configuration for the dataset
</ResponseField>

<ResponseField name="summary" type="string">
  Dataset summary
</ResponseField>

<ResponseField name="tags" type="string">
  Comma-separated tags
</ResponseField>

<ResponseField name="provisioner_state" type="object" nullable>
  Provisioner state information (for local dataset types)

  <Expandable title="properties">
    <ResponseField name="status" type="string">
      Provisioner status: `stopped`, `starting`, `running`, `stopping`, or `error`
    </ResponseField>

    <ResponseField name="state" type="object" nullable>
      Provisioner state including connection config and runtime info
    </ResponseField>

    <ResponseField name="started_at" type="string" nullable>
      ISO 8601 timestamp when provisioner was started
    </ResponseField>

    <ResponseField name="stopped_at" type="string" nullable>
      ISO 8601 timestamp when provisioner was stopped
    </ResponseField>

    <ResponseField name="error" type="string" nullable>
      Error message if status is `error`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of creation
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp of last update
</ResponseField>

<ResponseField name="connected_endpoints" type="array">
  List of endpoints connected to this dataset

  <Expandable title="items">
    <ResponseField name="id" type="string">
      Endpoint unique identifier
    </ResponseField>

    <ResponseField name="name" type="string">
      Endpoint name
    </ResponseField>

    <ResponseField name="slug" type="string">
      Endpoint URL slug
    </ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>
```bash cURL
curl -X POST https://your-domain.com/datasets/ \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "name": "legal-docs",
    "dtype": "local_file",
    "configuration": {
      "collectionName": "LegalDocuments",
      "httpPort": 8100,
      "ingestionPaths": [
        {
          "path": "/home/user/documents/legal",
          "description": "Legal documents directory"
        }
      ],
      "fileTypes": [".pdf", ".txt", ".docx"]
    },
    "summary": "Legal documents for RAG analysis",
    "tags": "legal,documents,analysis"
  }'
```

```bash Remote Weaviate
curl -X POST https://your-domain.com/datasets/ \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "name": "remote-knowledge-base",
    "dtype": "remote_weaviate",
    "configuration": {
      "http_url": "https://weaviate.example.com",
      "grpc_url": "grpc://weaviate.example.com:50051",
      "api_key": "your-weaviate-api-key",
      "collection_name": "KnowledgeBase",
      "content_property": "body",
      "metadata_properties": ["title", "author", "date"]
    },
    "summary": "Remote knowledge base for semantic search",
    "tags": "knowledge,remote,search"
  }'
```
</RequestExample>
